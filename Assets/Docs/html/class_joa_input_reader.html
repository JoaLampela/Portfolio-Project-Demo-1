void JoaInputReader.Attach ( PlayerInput playerInput)

Attaches this IInputReader to a specific PlayerInput instance and wires up the input action callbacks used by this component.

Parameters
playerInputThe PlayerInput that owns the per-instance InputActionAsset clone to read actions from.

If this IInputReader is already attached to another instance, it first calls Detach to unhook previous subscriptions and clear cached state.

Actions are resolved by GUID from the serialized InputActionReference fields (_moveAction, _lookAction, _jumpAction) against PlayerInput.actions.

The method subscribes to InputAction.performed and InputAction.canceled for each action, calls Enable to activate them, and marks the IInputReader as attached. Calling this method multiple times is safe; prior hooks are removed by Detach.

Typical usage from a join callback:

void OnPlayerJoined(PlayerInput playerInput)
{
IInputReader reader = playerInput.GetComponent<MonoBehaviour>().OfType<IInputReader>().FirstOrDefault()!;
reader.Attach(playerInput);
}
